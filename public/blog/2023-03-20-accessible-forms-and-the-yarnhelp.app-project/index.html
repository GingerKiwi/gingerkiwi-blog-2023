Accessible Forms and the "Yarn Help!" Website and PWA Project. https://yarnhelp.app

Mar 19, 2023

On Friday, March 17, I finally deployed Yarn Help! to the domain I purchased last October: https://yarnhelp.app. (What is currently deployed is the updates that I got done on Friday.) "Yarn Help!" is a site for knitters and crocheters of all levels who want yarn math help, as well as their friends and family that just want to know "how much yarn" to give them as a gift. Right now (March, 2023) the focus is on that page. Later on, you'll find yarn math help and tools for:

  • Calculating gauge
  • How many stitches to cast on for sweaters, socks, hats, and other projects
  • Weight equivalents for holding yarn double
  • Needle and hook size convertors
  • Sweater pattern generator for bottom up yoke pullovers based on Elizabeth Zimmerman's percentage system)
  • French language versions of some of the pages

Version 2.0: March 2023

The idea for this site started in March 2022, and was first published as a one page website with a really basic design in September, 2022. Currently, as of March 2023, development is focusing on a mobile friendly version of the "How Much Yarn page" followed by a responsive design of the entire site for larger screen sizes. I'll also be collaborating with a friend who's a senior dev to create a progressive web app version (pwa) of The Yarn Gift Helper page so people can install it on their phones and use it offline. (Some yarn shops have basements with spotty cell reception!)

  • View the current code and docs here: GitHub Repo for the Development Branch of YarnHelp.app

A11y: Accessibility

It's best practice to incorporate accessibility at all stages of the development process. For this quick version I've incorporated a colour scheme with good contrast, and that's not too bright.

Current UI Design

The UI theme is very simple for two reasons:

  1. It lets me focus on the logic and building out the site, without spending lots of time going down my favourite rabbit hole of design and css.
  2. It helps with load times, respects users data plans, and works better in spotty cell reception areas.

CSS

I've also included "prefers reduced motion", and left a commented section for "skip to main content". What was deployed on Friday is a mobile first design that just has the mobile version code. Media queries have to be added for larger screen sizes.

Note: If you're using a screenreader to read this post, the code for the screenshot is below the image. I haven't implemented code syntax in this blog yet.

Screenreader accessible code for image above:

/* -- A11Y -- */

  
/* Remove all animations, transitions, and smooth scrolling for people that prefer not to see them */
@media (prefers-reduced-motion: reduce) {
	html:focus-within {
	scroll-behavior: auto;
}
  
*,
*::before,
*::after {
	animation-duration: 0.01ms !important;
	animation-iteration-count: 1 !important;
	transition-duration: 0.01ms !important;
	scroll-behavior: auto !important;
	}
}
